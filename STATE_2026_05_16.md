## Verifiable security claims (each has a curl/cosign command)

The full matrix as of v37. Items marked **NEW since 2026-05-14** are shipped between the previous snapshot and this one.

| Claim | Verify with |
|---|---|
| Every `/v1/*` response carries `X-Cogos-Signature: hmac-sha256(hmac_secret, body)` | `curl -i https://cogos.5ceos.com/v1/models -H "Authorization: Bearer <key>"` |
| Every `/v1/*` response carries `X-Cogos-Attestation` Ed25519-signed receipt binding {req, resp, build, audit_chain_head} | Cookbook recipe at `/cookbook#verify-attestation` |
| Customer can use Ed25519 keypair auth (we store only the pubkey) | `POST /admin/keys` with `{"scheme":"ed25519",…}` |
| Per-tenant audit log hash-chained per-(tenant, app) | `GET /v1/audit?since=0` → response includes `chain_ok: true` |
| **NEW** Customer-sealed audit content (X25519 ECDH + AES-256-GCM) — server cannot decrypt rows it just wrote | `tests/sealed-audit.test.js` proves the math; production sealed rows visible via `/v1/audit` |
| **NEW** Public hash-chain head checkpoint, chained hourly | `curl https://cogos.5ceos.com/audit/checkpoint/latest` |
| **NEW** Checkpoint chain self-verifies | `curl https://cogos.5ceos.com/audit/checkpoint/verify` |
| Container image is cosignable | `cosign verify --key https://cogos.5ceos.com/cosign.pub cogos5ceos.azurecr.io/cogos-api:v<N>` |
| Cosign pubkey is publicly fetchable | `curl https://cogos.5ceos.com/cosign.pub` |
| Admin-key compare is constant-time | Source: `src/auth.js` uses `crypto.timingSafeEqual` |
| No live admin dashboard exposed | `curl -I https://cogos.5ceos.com/admin/live` → 404 |
| Scanner-target paths return canary content (case+slash variants covered) | `curl https://cogos.5ceos.com/.env` → fake env with AKIAIOSFODNN7EXAMPLE canary |
| Anomaly detector logs to data/anomalies.jsonl (shadow mode default) | `src/anomaly.js`; flip `ANOMALY_FAIL_CLOSED=1` for enforcement |
| Distroless runtime — no shell in container | `docker run --rm cogos5ceos.azurecr.io/cogos-api:v37 sh` → fails |
| HMAC + attestation key at rest envelope-encrypted under KV-resolved DEK | Source: `src/dek.js`, `src/keys.js`, `src/attestation.js` |
| Customer-driven key rotation with 24h grace | `POST /v1/keys/rotate` |
| Anomaly-driven auto-quarantine (scanner-pattern + valid-auth from same IP) | Source: `src/keys.js` quarantined_at field |
| **NEW** Honeypot hits persisted to `data/honeypots.jsonl` | `GET /admin/analytics/honeypots` returns real counts |
| **NEW** 429 events persisted to `data/rate-limits.jsonl` (5 distinct kinds) | `GET /admin/analytics/rate-limits` |

- `ANOMALY_FAIL_CLOSED` — flip to `1` after ~1 week of shadow-mode threshold calibration
- `RESEND_API_KEY`, `RESEND_FROM` — Resend fallback if SES is ever swapped out
- `COGOS_DEK_HEX`, `COGOS_DEK_FILE` — manual DEK override (otherwise lazy-generated to `data/.dek`)
- `COGOS_SESSION_SECRET`, `COGOS_MAGIC_LINK_SECRET` — manual secret override for dashboard sessions + magic links
- `COGOS_REVISION` — git commit SHA bound into attestation receipts (currently uses package.json version)
